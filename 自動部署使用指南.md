# 🚀 廣清宮快速記帳軟體 - 自動部署使用指南

## 📋 概述

本專案已配置完整的自動部署系統，支援以下兩種部署方式：

1. **GitHub Actions自動部署** - 推送程式碼後自動觸發
2. **本地一鍵部署** - 使用腳本快速部署

## 🔧 部署方式

### 方式一：GitHub Actions自動部署（推薦）

當您推送程式碼到 `main` 分支時，GitHub Actions會自動觸發部署：

```bash
git add .
git commit -m "更新內容"
git push origin main
```

**監控部署狀態：**
- 部署進度：https://github.com/sppwlkb/guangqing-temple-accounting/actions
- 網站地址：https://sppwlkb.github.io/guangqing-temple-accounting/

### 方式二：本地一鍵部署

#### Windows批次檔（推薦）
```cmd
# 雙擊執行
auto-deploy.bat

# 或在命令列執行
.\auto-deploy.bat
```

#### PowerShell腳本
```powershell
# 基本部署
.\auto-deploy.ps1

# 快速部署（跳過確認）
.\auto-deploy.ps1 -SkipConfirmation

# 自定義提交訊息
.\auto-deploy.ps1 -CommitMessage "修復bug並優化功能"
```

#### NPM腳本
```bash
# 基本部署（PowerShell）
npm run deploy

# 批次檔部署
npm run deploy:bat

# 快速部署（跳過確認）
npm run deploy:quick
```

## 📁 部署文件結構

自動部署會處理以下檔案：

```
dist/
├── index.html          # 主頁面
├── style.css           # 樣式文件
├── app.js              # 應用程式邏輯
├── firebase-config.js  # Firebase配置
├── cloud-sync.js       # 雲端同步功能
├── offline-storage.js  # 離線存儲功能
├── manifest.json       # PWA配置
├── service-worker.js   # 服務工作者
└── .nojekyll          # GitHub Pages配置
```

## ⚙️ 部署流程

### GitHub Actions流程
1. 檢出程式碼
2. 設置Node.js環境
3. 安裝依賴
4. 構建專案（如果有構建腳本）或複製靜態文件
5. 部署到GitHub Pages
6. 顯示部署結果

### 本地腳本流程
1. 檢查專案目錄
2. 檢查Git狀態
3. 自動提交未提交的變更
4. 推送程式碼到GitHub
5. 觸發GitHub Actions自動部署
6. 提供監控連結

## 🛠️ 故障排除

### 常見問題

#### 1. PowerShell執行政策錯誤
```powershell
# 解決方案：設置執行政策
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

#### 2. Git推送失敗
- 檢查網路連接
- 確認GitHub認證設置
- 檢查遠端倉庫地址

#### 3. GitHub Actions失敗
- 查看Actions日誌：https://github.com/sppwlkb/guangqing-temple-accounting/actions
- 檢查工作流程配置：`.github/workflows/deploy.yml`

#### 4. 網站無法訪問
- 確認GitHub Pages已啟用
- 檢查Pages設置：https://github.com/sppwlkb/guangqing-temple-accounting/settings/pages
- 等待幾分鐘讓更改生效

## 🔄 更新部署配置

如需修改部署配置，請編輯以下文件：
- `.github/workflows/deploy.yml` - GitHub Actions配置
- `deploy-config.json` - 部署設置
- `auto-deploy.ps1` / `auto-deploy.bat` - 本地部署腳本

## 💡 最佳實踐

1. **定期備份**：推送前確保重要變更已備份
2. **測試本地**：部署前在本地測試功能
3. **描述性提交**：使用清楚的提交訊息
4. **監控部署**：關注Actions執行狀態
5. **版本標記**：重要版本使用Git標籤
