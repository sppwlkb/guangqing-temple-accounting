# 🌐 宮廟獨立網址部署指南

## 🎯 為什麼需要獨立網址？

每個宮廟都應該有自己專屬的網址，原因如下：

- ✅ **品牌獨立性**：每個宮廟都有自己的特色
- ✅ **資料隔離**：確保各宮廟資料完全分離
- ✅ **專業形象**：提升宮廟的數位形象
- ✅ **SEO優勢**：更容易被搜尋引擎找到
- ✅ **成本控制**：避免共用資源的費用問題

## 🆓 方案一：GitHub Pages（推薦）

### 優點：
- ✅ 完全免費
- ✅ 自動SSL憑證
- ✅ 全球CDN加速
- ✅ 與GitHub完美整合
- ✅ 自動部署

### 網址格式：
```
https://[GitHub用戶名].github.io/[宮廟英文名]-temple/

實際範例：
廣清宮 → https://sppwlkb.github.io/guangqing-temple/
天后宮 → https://tianhou-admin.github.io/tianhou-temple/
關帝廟 → https://guandi-temple.github.io/guandi-temple/
```

### 🚀 快速建立步驟：

#### 使用自動產生器（推薦）
```powershell
# 執行專屬網站產生器
.\create-temple-site.ps1 -TempleName "廣清宮" -TempleEnglish "guangqing" -GitHubUsername "sppwlkb" -DPOName "李佳芬"
```

#### 手動建立步驟
1. **建立新的GitHub倉庫**
   - 倉庫名稱：`[宮廟英文名]-temple`
   - 設為Public
   - 不要勾選README

2. **複製並修改檔案**
   - 從主專案複製所有檔案
   - 修改title、宮廟名稱等資訊
   - 更新個資保護聯絡人

3. **上傳到新倉庫**
   ```bash
   git clone https://github.com/[用戶名]/[宮廟英文名]-temple.git
   # 複製修改後的檔案
   git add .
   git commit -m "初始化宮廟記帳系統"
   git push origin main
   ```

4. **啟用GitHub Pages**
   - Settings → Pages
   - Source: Deploy from a branch
   - Branch: main 或 gh-pages

## 🌟 方案二：自訂網域

### 優點：
- ✅ 最專業的形象
- ✅ 最容易記憶
- ✅ 最佳SEO效果
- ✅ 完全的品牌控制

### 網址格式：
```
https://[宮廟英文名]-temple.com/

實際範例：
廣清宮 → https://guangqing-temple.com/
天后宮 → https://tianhou-temple.com/
關帝廟 → https://guandi-temple.com/
```

### 🛠️ 設定步驟：

1. **購買網域**
   - 推薦：Namecheap, GoDaddy, Cloudflare
   - 費用：約NT$ 500-1000/年
   - 選擇：`[宮廟英文名]-temple.com`

2. **設定DNS**
   ```
   類型: CNAME
   名稱: www
   值: [GitHub用戶名].github.io
   
   類型: A
   名稱: @  
   值: 185.199.108.153
   值: 185.199.109.153
   值: 185.199.110.153
   值: 185.199.111.153
   ```

3. **在GitHub設定自訂網域**
   - Settings → Pages
   - Custom domain: 填入您的網域
   - 等待DNS驗證完成

## ⚡ 方案三：Netlify

- ✅ 部署速度快

- ✅ 函數支援

4. 可選：連接自訂網域

  -TempleName "廣清宮" \
